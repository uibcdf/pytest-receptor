# External Test Paths Produce an Invalid Rerun Command and Location

**Reported:** 2026-07-18  
**Pilot:** MolSysMT shadow evaluation  
**Category:** Diagnostic sufficiency

## Summary

When pytest is invoked from the MolSysMT repository with a test file outside
the pytest root directory, pytest-receptor strips too much path context. The
rendered rerun command cannot locate the test from the original working
directory, and the primary crash location contains a duplicated repository
component.

## Environment

- Python 3.13.12
- pytest 9.0.2
- pytest-receptor 0.6.0, editable installation
- working directory: `/home/diego/repos@uibcdf/molsysmt`
- MolSysMT addopts: `--doctest-modules -q --import-mode=importlib`
- reproduces with `-n 0` and `-n 12`

## Invocation

```text
python -m pytest --receptor=llm --receptor-stats -n 12 /tmp/molsysmt_receptor_pilot/test_fixture_cascade.py
```

Relevant receptor output:

```text
[1] AttributeError | 20 tests | setup
    molsysmt/molsysmt/__init__.py:131
    AttributeError: module 'molsysmt' has no attribute 'MolSys'
    frames: test_fixture_cascade.py:9 -> molsysmt/__init__.py:131
    ...
    rerun: pytest test_fixture_cascade.py -q
```

There are two path problems:

1. From the original working directory, the suggested rerun exits with status
   4:

   ```text
   $ pytest test_fixture_cascade.py -q
   ERROR: file or directory not found: test_fixture_cascade.py
   ```

2. `molsysmt/molsysmt/__init__.py:131` does not resolve from the original
   working directory. The usable package path is
   `molsysmt/__init__.py:131`, which the `frames:` line already renders
   correctly.

## Expected behavior

Every rendered path and rerun command should be executable or resolvable from
the directory where pytest was invoked. For this invocation, a sufficient
rerun would retain the absolute test path:

```text
rerun: pytest /tmp/molsysmt_receptor_pilot/test_fixture_cascade.py -q
```

Alternatively, the report could state a working directory explicitly and make
all paths relative to it. The important contract is that copying the command
verbatim reproduces the selected occurrences.

The primary crash location should use the same normalized, resolvable path as
the frame chain:

```text
molsysmt/__init__.py:131
```

## Impact

- The advertised exact rerun command is not exact for external paths.
- An agent following it immediately spends another pytest invocation on a path
  error unrelated to the original failure.
- The duplicated source path can force the agent to search for the actual file,
  violating the pilot's stdout-sufficiency criterion.
- Temporary regression tests and tests generated by agents commonly live
  outside the repository, so this is relevant to agent-driven workflows rather
  than an artificial path shape.

## Suggested regression coverage

Exercise serial and xdist invocations for test targets that are:

- relative paths inside the pytest root;
- absolute paths inside the pytest root;
- absolute paths outside the pytest root;
- paths containing spaces;
- individual external node IDs.

For every case, execute the rendered rerun command from the original working
directory and assert that it collects the intended occurrences. Also assert
that every rendered source location resolves from that same directory.

---

## Resolution

**Fixed 2026-07-18.** Both symptoms had one cause: paths were rendered relative
to `rootpath`, and rootdir is not where pytest was invoked. Naming a test outside
the project sets rootdir to the common ancestor, so everything printed resolved
from nowhere.

The duplication was a second-order effect. When the relative path escaped
rootdir, the fallback kept the last three components of the absolute path --
which duplicates whenever a package shares its name with its repository
directory, hence `molsysmt/molsysmt/__init__.py`.

Every printed path is now relative to `config.invocation_params.dir`, so it
resolves from the shell the reader is in. That covers the crash location, the
`frames:` chain, the rerun command, and the node IDs in the occurrence list --
the last of which the report did not mention, but which had the same defect and
is equally likely to be copied.

Dependency paths still render in short form, since recognising the library
matters more than resolving a file nobody will open.

A regression asserts the rerun command's target exists from the invocation
directory, and another asserts a package sharing its repository's name is never
rendered twice.
